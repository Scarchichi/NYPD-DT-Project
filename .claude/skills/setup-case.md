# Setup Case

Conversational setup that configures the case study project by asking questions and writing files programmatically.

## Usage

```
/setup-case
```

## Instructions

### Pre-flight

1. **Check if already configured**: Read `case-config.yaml`. If values are NOT the defaults ("Company Name", "Topic Description", etc.), warn the user:
   ```
   This project is already configured for [company_name].
   Running /setup-case again will overwrite your configuration.
   Do you want to continue? (yes/no)
   ```
   If user says no, stop. If yes, proceed.

### Conversational Setup

2. **Ask questions one at a time.** Do NOT present a form or ask all at once. Wait for each answer before asking the next. Use this sequence:

   **Question 1**: "What company or organization is this case study about? Give me the full legal name and a short name for file naming."
   - Extract: `company_name` (full) and `company_short` (short)
   - Example: "Moderna, Inc." → full: "Moderna, Inc.", short: "Moderna"

   **Question 2**: "In one sentence, what is the case study topic?"
   - Extract: `topic`
   - Example: "AI-Driven Drug Discovery and mRNA Platform Strategy"

   **Question 3**: "Who is the protagonist — the primary decision-maker the case follows? Give me their name and title."
   - Extract: `protagonist_name` and `protagonist_title`
   - Example: "Stéphane Bancel, CEO"

   **Question 4**: "What course is this for? I need the course name/number, institution, and semester."
   - Extract: `course_name`, `institution`, `semester`
   - Example: "ITEC-617, Kogod School of Business, Spring 2026"

   **Question 5**: "Is this a business case or a public policy case?"
   - Extract: `case_type` — values: "business" or "public_policy"
   - Default: "business"

   **Question 6**: "How long is the classroom session in minutes? (typical: 75-90)"
   - Extract: `session_length_minutes`
   - Default: 80

### Write Files

3. **Write `case-config.yaml`** with the collected values:

   ```yaml
   # Auto-generated by /setup-case. You do not need to edit this manually.
   # To reconfigure, run /setup-case again.

   # CASE INFORMATION
   case:
     company_name: "{company_name}"
     company_short: "{company_short}"
     topic: "{topic}"
     protagonist_name: "{protagonist_name}"
     protagonist_title: "{protagonist_title}"
     case_type: "{case_type}"

   # COURSE INFORMATION
   course:
     name: "{course_name}"
     institution: "{institution}"
     semester: "{semester}"

   # DOCUMENT SETTINGS
   documents:
     target_word_counts:
       main_case: 5000
       supplement: 3000
       teaching_note: 3000
       additional_sources: 4000
     session_length_minutes: {session_length_minutes}

   # REPOSITORY SETTINGS
   repository:
     private: true
     license: "CC-BY-NC-4.0"

   # TEMPLATE METADATA (do not modify)
   template:
     version: "3.1.0"
     source: "https://github.com/leifulstrup/Agentic-AI-Case-Study-Development-Starter-Kit"
   ```

4. **Write `PROJECT_CONTEXT.md`** with actual values:

   ```markdown
   # Project Context

   Quick reference for session continuity. Auto-maintained by skills — you do not need to edit this manually.

   ## Project Overview

   **Case Study**: {company_short} — {topic}
   **Protagonist**: {protagonist_name}, {protagonist_title}
   **Course**: {course_name} at {institution}, {semester}
   **Case Type**: {case_type}
   **Session Length**: {session_length_minutes} minutes

   ## Current Status

   **Phase**: Setup Complete — Ready for Source Collection

   **Documents**:
   - [ ] Additional Sources — not started
   - [ ] Main Case — not started
   - [ ] Supplement — not started
   - [ ] Teaching Note — not started

   ## Source Quality

   **Source Registry**: Not yet populated (run `/add-sources`)
   **Last Assessment**: None

   ## Verification Debt

   **Open Items**: 0
   See `verification-debt.yaml` for details.

   ## Key Decisions Made

   - Case type: {case_type}
   - Session length: {session_length_minutes} minutes

   ## Open Questions

   - Source materials needed

   ## Next Steps

   1. Add source materials to `sources/` folder
   2. Run `/add-sources` to register them
   3. Run `/assess-sources` to evaluate readiness
   ```

5. **Write `verification-debt.yaml`** (if it doesn't exist):

   ```yaml
   # Verification Debt Tracker
   # Auto-maintained by /write-document and verification skills.
   # Do not edit manually.
   #
   # Each item tracks an unverified AI-generated claim that needs
   # a source or correction before publication.
   #
   # Statuses: unverified | verified | removed | flagged
   last_updated: "{today's date}"
   items: []
   ```

6. **Create `sources/Source_Registry.md`** (if it doesn't exist) using the template from the Source_Registry skill.

### Confirmation

7. **Display summary table and next step**:

   ```
   ## Case Study Configured

   | Field | Value |
   |-------|-------|
   | Company | {company_name} ({company_short}) |
   | Topic | {topic} |
   | Protagonist | {protagonist_name}, {protagonist_title} |
   | Course | {course_name} at {institution} |
   | Semester | {semester} |
   | Case Type | {case_type} |
   | Session | {session_length_minutes} minutes |

   ### Files Written
   - `case-config.yaml` — project configuration
   - `PROJECT_CONTEXT.md` — session continuity
   - `verification-debt.yaml` — verification debt tracker
   - `sources/Source_Registry.md` — source registry (if created)

   ### Next Step
   Add your source materials (transcripts, articles, financial reports) to the
   `sources/` folder, then run `/add-sources` to register them.

   Or say "Help me find sources for this case" and I'll suggest where to look.
   ```

## Notes

- This skill replaces `/setup-project` from v2
- The user never needs to open or edit `case-config.yaml` directly
- If the user provides partial info (e.g., just company name), ask follow-up questions for the rest
- Accept natural language answers and parse intelligently (e.g., "ITEC-617 at Kogod, Spring 2026" → course_name: "ITEC-617", institution: "Kogod School of Business", semester: "Spring 2026")

## Output

Display the summary table directly to the user. No log file needed.
